# 小白友好发布 / 下载 / 加载 — 设计文档

> **日期**：2026-06-14
> **目标**：让不懂技术的 VC/PE/FA 群友也能顺畅地发布和消费 Skill 内容。
> **原则**：复杂的部分我们自动帮做，用户只面对大白话和具体动作。

## 一、问题诊断

群友画像：基本不懂技术，平时用 ChatGPT/Claude 主要是聊天框粘贴 prompt。现状发布的摩擦点：

1. **行话劝退**：第一屏就是「Skill 类型 / Prompt / Agent Skill / SKILL.md / Workflow / Tool Stack」。
2. **表单太长**：5 个 Section + 一堆必填，吓退只想分享一个小东西的人。
3. **场景标签是知识负担**：「工作场景」10 选 1 必填，且是英中混排行话。
4. **正文/附件/安装说明内容重叠**：小白不知道一段 prompt 该写进哪。
5. **消费侧断层**：下载后不知道拿文件干嘛，没有「怎么用」引导。

## 二、核心方案

把发布从「一次性 5 段长表单」改成「第一问决定后面显示什么」的问导式三分支流程。配套：行话→大白话、场景标签加示例、下载加「怎么用」固定说明、提示词加一键复制。**不动数据模型、不动后端 API，纯前端 + 文案。**

## 三、问导式发布流

### 首屏三分支（按"大家嘴里怎么说"分）

| 分支 | 卡片名 | 副标题 | 自动映射 assetType |
|------|------|------|------|
| A | 提示词 | 一段复制就能用的 AI 指令 | 固定 `prompt` |
| B | Skill 文件 | 模板 / 表格 / 脚本 / SKILL.md 等现成文件 | 用户选大白话类型，映射见下 |
| C | 方法论 | 完整的工作流程或工具组合方案 | 用户选，映射见下 |

### 分支字段

**A · 提示词**：标题 → 提示词大文本框（存进 body）→ 场景标签（带示例）→ 折叠补充标签
**B · Skill 文件**：标题 → 上传文件（最显眼）→ "这是什么类型的文件？"（模板/表格/脚本/SKILL.md/PPT）→ 选填介绍 → 场景标签 → 折叠补充标签
**C · 方法论**：标题 → 富文本正文 → "这属于哪类？"（工作流程/工具组合/实战案例）→ 选填附件 → 场景标签 → 折叠补充标签

### assetType 映射（前端表）

- 分支 B：模板→`template`、表格→`template`、脚本→`api-script`、SKILL.md→`agent-skill`、PPT→`template`
- 分支 C：工作流程→`workflow`、工具组合→`tool-stack`、实战案例→`case-study`

### 场景标签加示例（解决 10 选 1 负担）

每个标签配一句小白能秒懂的示例（如「初筛判断 —— 刚拿到一个项目，快速决定要不要深看」），标签名中文化、去英中混排。

## 五、详情页 · 消费体验

### 「怎么用」固定说明条

附件区 / 正文区上方，根据 assetType 自动显示一句固定模板（前端写死，零维护）：

- `prompt`：复制正文里的提示词，粘贴到 ChatGPT / Claude / Kimi 等任意 AI 对话框，把方括号 `[像这样]` 的部分换成你的实际内容。
- `template`：下载文件后用 Excel / Word / PPT 打开（看文件后缀），按里面的说明填写。建议先复制一份原文件再改，保留模板。
- `agent-skill`：这是个 SKILL.md 文件。在 Claude Code 里放到 `~/.claude/skills/` 目录下即可调用。（看不懂这步？先用其他类型就好）
- `api-script`：下载脚本文件后，按正文里的"运行步骤"配置环境。需要会一点点命令行，不懂可以评论区问作者。
- `workflow`：正文里是完整的工作流程，按步骤照做即可。
- `tool-stack`：正文里推荐了工具组合，点链接去各工具官网注册/下载，按顺序组合使用。
- `case-study`：这是一个完整案例，建议先通读一遍理解思路，再对照自己的项目套用。

### 提示词一键复制

仅 `prompt` 类型：正文区右上角「复制」按钮，一键复制全文。

### 可见性策略

提示词全文免费可看（价值先可见），「复制」按钮需登录（引导注册）。

## 八、验证标准

1. `tsc --noEmit` 0 错误
2. 重走发帖全闭环：三个分支都能发成功、都能在首页 Feed 和 Skills Map 出现
3. 提示词分支：详情页有「怎么用」+「复制」按钮
4. Skill 文件分支：详情页附件区有「怎么用」+「下载」
5. 行话已全部换成大白话（grep 检查无残留）
