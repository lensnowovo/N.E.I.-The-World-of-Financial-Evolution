# 投稿、评论、举报与版权处置 SOP

## 投稿

1. 投稿人确认原创/授权/开源/公共领域类型，填写来源、许可证和版本。
2. 前端提示先脱敏；禁止 BP、财务底稿、投委会材料、录音、客户/LP 名单和商业秘密。
3. 机器扫描失败或发现风险时必须进入人工待审，不能 fail-open。
4. 人工检查违法内容、版权、泄密、恶意指令、外链、附件和 AI 声明。
5. Web 发布与 MCP 准入是两个独立决策；编辑后重新审核并撤销旧 MCP 准入。

## 评论

- 在完成真实身份认证和评论审核能力前，建议阶段性关闭公开评论，或全部进入待审。
- 必须支持举报评论、用户和帖子，并保存处理人、理由、时间、前后状态和申诉结果。

## 版权通知

- 接收权利人身份/联系方式、URL、权利证明、具体说明和真实性声明。
- 高风险内容先暂停展示和 MCP 分发，再通知投稿人举证。
- 投稿人可提交原创记录、授权书、许可证和版本证据。
- 最终结果记录恢复、补充来源、修改、继续下架或移交处理。

## 时限建议

- 泄密、恶意代码、违法金融营销：立即下架并升级安全事件。
- 材料完整的版权投诉：1 个工作日内临时处置，5 个工作日内给出阶段结果。
- 普通举报：3 个工作日内初审；复杂情况告知延期。
