# 安全与个人信息事件响应

## 触发

- 账号、数据库、对象存储、MCP Token、Memory 许可证或密钥疑似泄露。
- 违法/侵权/保密材料公开扩散。
- 管理员权限异常、批量抓取、恶意上传、服务被攻击或数据损毁。

## 前 30 分钟

1. 记录发现时间、报告人、影响服务和当前证据，不在公开群传播敏感正文。
2. 撤销泄露 Token/密钥，冻结可疑账号或接口；必要时只读/下线相关功能。
3. 保存 Vercel、数据库、对象存储和管理操作日志，禁止无记录清库。
4. 确定事件负责人和单一沟通渠道。

## 前 4 小时

1. 判断数据种类、用户数量、地域、是否含敏感/未公开项目材料。
2. 评估事件等级及网信、公安、行业主管部门报告义务。
3. 达到法定情形时按《国家网络安全事件报告管理办法》等要求报告；不能确定时联系属地主管部门。
4. 准备用户通知：发生了什么、可能影响、已采取措施、用户应做什么、联系渠道。

## 恢复与复盘

- 验证攻击路径关闭后分阶段恢复。
- 形成时间线、根因、影响、处置、通知、数据删除/恢复和长期修复清单。
- 事件证据只向必要人员开放，按法律和争议需要保存。
- 将修复转为回归测试、监控和管理员审计规则。

公开沟通不得包含密钥、完整 Token、数据库连接串、用户邮箱、项目名称或受影响材料正文。
