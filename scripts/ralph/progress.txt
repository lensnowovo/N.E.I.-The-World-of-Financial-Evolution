## Codebase Patterns
- MCP 工具鉴权：`app/api/mcp/route.ts` 通过 `getUidFromRequest(req)` 解析 Bearer token（sha256 哈希后比对 `User.mcpTokenHash`），uid 必须通过 per-request 闭包传入 tool handlers，**绝不**用模块级全局变量（否则并发请求会串号）
- 本项目无独立 typecheck script，统一用 `npx tsc --noEmit` 做静态校验；改完代码只需跑这一条
- `createMcpHandler(initializeServer)` 的 `initializeServer` 回调在每次 POST 请求都会被调用（内部为每请求新建 McpServer + transport），所以在 POST/GET/DELETE 入口处按 uid 重建 handler 是安全的，且天然形成闭包隔离
- `sanitizeHtml`（lib/validate.ts）走「白名单标签 + 每标签属性白名单（a.href / img.src / img.alt）+ URL scheme 过滤」；post.body 在写入(POST /api/posts) 与渲染(app/posts/[id]/page.tsx 的 safeBody) 两层都过，改 body 渲染/编辑接口时务必维持此不变量
- rate limiting：所有限流收敛到 `lib/rate-limit.ts` 的 `checkRateLimit(key, limit, windowMs)`（纯内存固定窗口，无依赖，懒过期 + 60s 周期清理）；返回 `retryAfter` 单位是毫秒，写 HTTP Retry-After 头需 Math.ceil(/1000)。key 自带命名空间如 `verify-email:email:${email}`
- 客户端 IP：用 `lib/rate-limit.ts` 的 `getClientIp(req)`（纯 header 解析，无 next 依赖），取 `x-forwarded-for` 第一项，兜底 `x-real-ip`，缺失返回 `'unknown'`（共享桶，比不限流安全）
- rate-limit 维度与限额约定：verify-email = email 3/10min + IP 10/10min；login = email **失败** 5/15min（只在认证失败路径计数，成功登录不计）；register / reset-password = IP 10/10min。429 响应统一带 `Retry-After` 头（秒）+ body `{ error, retryAfter }`
- OAuth/CSRF state：收敛到 `lib/oauth-state.ts` 的 `issueOAuthState()`/`verifyOAuthState()`（HMAC 签名 token 复用 SESSION_SECRET，短期 one-shot cookie `pevc_oauth_state`，SameSite=Lax）。需要做 CSRF 防护的第三方跳转回跳场景都走这套
- 开放重定向防护：客户端/服务端拿 `?next=` 做跳转前必须校验「单个 `/` 开头 && 非 `//` && 不含 `\`」。`//evil.com` 是协议相对 URL，浏览器会当 https://evil.com；部分浏览器会把 `/\` 规范化成 `//`，所以反斜杠也要拒。不合规一律回退 `/`
- 用户枚举模糊：登录/重置/发码等鉴权接口，账号不存在时不能返回 "账号不存在"/"未注册"，必须返回与「凭证错误」完全一致的文案 + 状态码（如密码模式 → "邮箱或密码错误" 401；验证码模式 → "验证码无效或已过期" 400）。否则攻击者可枚举探测已注册邮箱
- 管理员判断：用 `getCurrentUser()` 返回的 `isAdmin` 字段（lib/session.ts select 已含）；首批 admin 通过 `ADMIN_BOOTSTRAP_EMAILS` env（逗号分隔）引导，收敛到 `lib/admin.ts` 的 `shouldBootstrapAdmin(email)`，仅在用户 **创建** 路径（register/github-callback 新建/seed）调用 —— 已存在用户登录不应被升级为 admin
- 软删除过滤：Post.deletedAt 是软删标记（null = 活，非 null = 已删）。所有公开 Post 查询必须过滤 `deletedAt: null`：findMany/count/groupBy 直接进 `where`；findUnique 走 `where: { id }` 不能加非唯一字段，改为「fetch 后 `|| post.deletedAt` 当作未找到」；若 findUnique 用了 `select`（非 include），必须把 `deletedAt: true` 加进 select 否则 tsc 报字段不存在。列表查询统一收敛到 `lib/feed.ts` 的 `buildFeedWhere`（已带过滤）；owner-only 视图（profile posts tab owner 分支 / dashboard 收藏列表）按需求可不过滤，让作者看到自己删掉的
- 管理员 API 守卫：收敛到 `lib/auth-guard.ts` 的 `requireAdmin()`，返回 `{ user: SessionUser } | NextResponse`。调用方 `const g = await requireAdmin(); if (g instanceof NextResponse) return g;` 后即可在 g.user 上拿 admin 身份。语义：未登录 → 401（与项目其他写接口一致），已登录非管理员 → 403。isAdmin 来自 getCurrentUser select，守卫本身不查 DB
- 详情页 action 入口（编辑/举报）放在 header meta 行的右半 `flex items-center gap-3 shrink-0` 中，按 canEdit / uid 条件渲染。客户端组件（ReportButton/PostActions）作为子组件插入；server page 负责鉴权 + 数据，client component 负责交互态
