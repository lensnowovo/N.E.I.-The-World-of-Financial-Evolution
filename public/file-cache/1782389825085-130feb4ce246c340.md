# AI Native SaaS 投资研究框架

## 这是什么
AI 原生 SaaS（AI Agent / AI 应用 / AI 基础设施）的投资分析框架。SaaS 指标体系是这套行业的**通用语言**——ARR/NRR/Rule of 40/Magic Number 是 VC 尽调的标配。但 2026 年的关键变化是：**AI 对 SaaS 行业产生分化**——按席位计费的传统 SaaS 受 AI 冲击，按使用量计费的 AI 基础设施 SaaS 反而受益于 Agent 浪潮。适合投资 AI 应用层、Agent 平台、AI Native 基础设施的 PE/VC。

## 行业研究框架（6 维度）

### 维度 1：SaaS 核心指标体系（2026 版）

> 2026 年的核心转变：从"不惜代价的增长"转向"高效增长"——投资人最看重留存质量、回收周期、烧钱效率。

#### 收入类指标
| 指标 | 定义 | 说明 |
|---|---|---|
| **MRR（月经常性收入）** | 月度订阅收入（剔除一次性费用/专业服务） | 若混入专业服务，所有下游指标失真 |
| **ARR（年经常性收入）** | MRR × 12 | ARR > 100 万美元后对话从 MRR 转向 ARR |
| **ARPU（每用户平均收入）** | MRR ÷ 活跃付费客户 | ARPU 上升 = 向高端市场移动 |
| **ARR Bridge（变动表）** | 期初+新增+增购-缩减-流失 = 期末 | B 轮以后尽调必要求 |

#### 获客类指标
| 指标 | 定义 | 2026 基准 |
|---|---|---|
| **CAC（客户获取成本）** | 全额 S&M 费用 ÷ 新客户数 | 需含人力成本，否则严重低估 |
| **CAC 回收期** | CAC ÷ (ARPA × 毛利率) | < 12 月优秀；12-18 月合格；> 24 月危险 |

#### 留存类指标（**投资人最看重**）
| 指标 | 定义 | 2026 基准 |
|---|---|---|
| **月客户流失率（Logo Churn）** | 流失客户 ÷ 期初客户 | < 1% 优秀；< 3% 可接受；> 5% 严重 |
| **NRR（净收入留存率）** | (期初+增购-缩减-流失) ÷ 期初 | **> 110% 健康；> 125% 世界级（Snowflake IPO 前达 158%）** |
| **LTV** | (ARPA × 毛利率) ÷ 流失率 | 早期公司上限设 3-4 年 |

#### 资本效率类指标（**2026 决定性指标**）
| 指标 | 定义 | 2026 基准 |
|---|---|---|
| **Burn Multiple** | 净烧钱 ÷ 新增 ARR | < 1 卓越；1-1.5 高效；2-3 低效；> 3（ARR > 1000 万时）严重 |
| **Magic Number** | (季度新增 ARR × 4) ÷ 上季度 S&M | > 1 加大投入；0.7-1 健康中位数；< 0.5 重新审视 GTM |
| **Rule of 40** | ARR 增速% + 利润率% | **≥ 40 底线；≥ 50 溢价估值门槛；≥ 60 估值倍数 2-3x 同行** |

### 维度 2：Rule of 40 实战应用

**核心逻辑**：SaaS 公司可以高增长低利润，也可以低增长高利润，但**两者之和不能低于 40**。

**真实公司对照（2026Q1）**：

| 档位 | 公司 | 增速 | 利润率 | Rule of 40 | PS(TTM) |
|---|---|---|---|---|---|
| 🏆 顶级 | Palantir | 35% | 53% | **88** | 63x |
| 🏆 顶级 | CrowdStrike | 32% | 28% | **60** | 20x |
| 🏆 顶级 | Datadog | 32% | 25% | **57** | 15.5x |
| ✅ 达标 | ServiceNow | 22% | 24% | **46** | 10.8x |
| ✅ 达标 | MongoDB | 28% | 14% | **42** | 11.5x |
| ❌ 不达标 | Snowflake | 25% | 11% | **36** | 10.6x |
| ❌ 不达标 | Salesforce | 11% | 17% | **28** | 4.1x |
| ❌ 不达标 | HubSpot | 18% | 10% | **28** | 6.9x |

**A 股 / 港股对照**：仅金山办公（46）和迈富时（44）达标；金蝶（30）/用友（-12）/微盟（25）/有赞（-15）均不达标。

### 维度 3：AI 对 SaaS 的分化逻辑（2026 最关键洞察）

**分水岭：按使用量计费 vs 按席位计费**

#### ❌ 受 AI 冲击（按席位计费）
```
AI 让每个员工完成更多工作 → 需要的席位减少 → 收入下降
```
**压力名单**：Zoom、Docusign、HubSpot、Salesforce（传统通信层）。

#### ✅ 受益于 AI Agent（按使用量计费）
```
Agent 1 个指令 → N 个推理步骤 × M 个工具调用 = 指数级查询量
→ 数据库查询/日志/通信量不减反增 → 账单自动增加
```
**Gartner 数据**：多 Agent 系统查询量从 2024Q1 到 2025Q2 增长 **1,445%**。

**受益名单（按使用量计费的数据基础设施）**：

| 公司 | 受益逻辑 | Rule of 40 |
|---|---|---|
| **Datadog** | Agent 每步推理产生 logs + traces | 57 ✅ |
| **MongoDB** | Agent 状态变更需事务型数据库 | 42 ✅ |
| **Confluent** | 批→实时架构迁移（预期差最大） | — |
| **Snowflake** | Agent 查询历史数据做决策 | 36 ❌ |
| **Twilio** | Agent 完成需通信通知 | 23 ❌（转型中）|

### 维度 4：AI Agent 价值链六层结构

| 层次 | 功能 | AI Native 竞争态势 |
|---|---|---|
| **L0 大模型** | 推理引擎 | ⭐ 商品化压力（OpenAI/Anthropic/Google） |
| **L1 Agent 编排** | 任务编排框架 | ⭐⭐⭐ 需求真实但竞争混乱（ServiceNow/Salesforce） |
| **L2 数据库与存储** ⭐ | 数据持久化 | ⭐⭐⭐⭐⭐ **最 solid**（MongoDB/Snowflake） |
| **L3 实时数据流** | 批→实时迁移 | ⭐⭐⭐⭐ 预期差最大（Confluent） |
| **L4 可观测性** ⭐ | 监控/追踪/成本 | ⭐⭐⭐⭐⭐ **最 solid**（Datadog） |
| **L5 安全与权限** | Agent 权限管控 | ⭐⭐⭐ 重要（CrowdStrike/Zscaler） |
| **L6 通信与交互** | Agent 通知/协作 | ⭐⭐⭐⭐ 转型按 Agent 收费（Twilio） |

### 维度 5：分阶段指标侧重

| 阶段 | ARR | 重点指标 |
|---|---|---|
| **种子轮** | < 100 万 | MRR 环比（~15%）、客户流失率、定性 NPS |
| **A 轮** | 100 万-1000 万 | ARR 增速（200%→100%）、NRR（≥110%）、CAC 回收期（< 18 月）、Burn Multiple（< 2x）、毛利率（> 70%） |
| **B 轮+** | > 1000 万 | **Rule of 40（> 50%）**、Burn Multiple（< 1.5x）、NRR（≥115%）、Magic Number（> 0.7）、同期群留存曲线 |

### 维度 6：AI Native SaaS 的独特关注点

1. **API/推理成本占比**：AI SaaS 的"毛利率陷阱"——大模型 API 成本可能占收入 20-40%，压低真实毛利率至 50-60%（而非传统 SaaS 的 75%+）。需用"扣除推理成本后的毛利率"重新评估。
2. **数据飞轮**：用户使用越多 → 数据越多 → 模型越好 → 用户越多。判断数据是否真正回流到模型优化。
3. **模型依赖风险**：依赖单一模型（如 GPT-4）的 AI SaaS，模型供应商提价或停止服务即被动。模型无关性（Model-agnostic）是壁垒。
4. **从订阅制到按结果/用量计费**：AI Agent 的趋势是按"任务完成数"或"用量"计费——收入波动性增大，但天花板更高。
5. **"包装型" vs "原生型"AI SaaS**：在现有 SaaS 上套壳 AI（易被替代）vs 从第一天就以 AI 为核心设计（真壁垒）。

## 行业核心指标速查

| 指标 | 公式 | 优秀 | 合格 | 危险 |
|---|---|---|---|---|
| **ARR 增速** | (本年-上年)/上年 | > 100% | 30-50% | < 20% |
| **NRR** | (期初+增购-缩减-流失)/期初 | > 120% | > 100% | < 90% |
| **毛利率（传统 SaaS）** | (收入-COGS)/收入 | > 80% | > 70% | < 60% |
| **毛利率（AI SaaS，扣推理成本）** | 同上但 COGS 含 API 成本 | > 65% | > 50% | < 40% |
| **CAC 回收期** | CAC/(ARPA×毛利率) | < 12 月 | < 18 月 | > 24 月 |
| **LTV/CAC** | LTV/CAC | > 5 | 3-5 | < 3 |
| **Burn Multiple** | 净烧/新增ARR | < 1 | < 2 | > 3 |
| **Magic Number** | (季增ARR×4)/上季S&M | > 1 | 0.7-1 | < 0.5 |
| **Rule of 40** | 增速%+利润率% | > 60 | > 40 | < 30 |
| **SBC 占比** | 股权激励/收入 | < 10% | < 20% | > 25% |

## 壁垒判断

### 真壁垒（AI Native SaaS）
1. **按使用量计费 + 数据不可迁移**：Agent 状态数据存入某数据库后，切换成本极高（锁定效应）。
2. **模型无关性**：无论 OpenAI/Claude/Gemini 谁赢，基础设施层都赢。
3. **数据飞轮**：用户使用数据真正回流到模型优化——形成正循环。
4. **垂直行业 know-how + 数据资产**：通用 AI 工具易被复制，垂直行业（法律/医疗/金融）的专有数据是壁垒。
5. **网络效应**：双边市场（如 Agent 市场）/ 多方协作平台。

### 假壁垒（警惕）
1. **"AI 包装"型 SaaS**：在现有工作流上套一层 GPT API——6 个月内可被复制。
2. **纯依赖单一模型**：无自有模型/数据——模型供应商提价即利润消失。
3. **高增长但 NRR < 100%**：靠获客堆增长，留存漏水——Burn Multiple 会暴露。
4. **"覆盖多行业"但无深耕**：通用 AI 工具无差异化。

## 红旗识别
1. **NRR < 100% 且持续下滑**——客户在流失，增长靠获客硬撑。
2. **Burn Multiple > 3（ARR > 1000 万时）**——资本效率问题，绝无例外。
3. **Rule of 40 < 30%**——要么增长不够，要么利润不够，通常两者都不够。
4. **毛利率 < 50% 且主因是 AI API 成本**——未解决推理成本问题。
5. **CAC 回收期 > 24 月 + 毛利率 < 75%**——借用未来收入资助今天增长。
6. **ARR Bridge 中"流失"项持续 > "增购"**——产品价值衰减。
7. **SBC 占比 > 25%**——真实盈利能力被股权激励稀释（如 Databricks 25%、GitLab 22%）。
8. **单一模型依赖 + 无模型切换能力**——被供应商锁死。
9. **"AI 功能"而非"AI Native"**——套壳产品，6 个月淘汰期。
10. **Magic Number < 0.5 且无改善**——GTM 策略失效。

## 占位符
- `[产品类型]`：AI Agent / AI 基础设施 / AI Native 应用 / AI 增强 SaaS。
- `[计费模式]`：按席位 / 按使用量 / 按结果。
- `[ARR / ARR 增速]`。
- `[NRR]`：最近 12 个月数据。
- `[Rule of 40 分解]`：增速% + 利润率%。
- `[毛利率（传统 vs 扣推理成本）]`。
- `[Burn Multiple / Magic Number]`。
- `[ARR Bridge]`：期初 + 新增 + 增购 - 缩减 - 流失 = 期末。
- `[模型依赖度]`：依赖哪几个模型 + 切换能力。
- `[数据飞轮验证]`：用户数据是否回流到模型优化。
