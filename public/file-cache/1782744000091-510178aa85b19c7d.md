# Skill：PE/VC 投研 MCP & API 工具箱

> **一句话定位**：一份按投研工作流组织的 MCP/API 目录——不是技术百科，而是"遇到什么信息需求，该接哪个 MCP/API"的实战选型指南。

---

## 一、这是什么

MCP（Model Context Protocol）是 Anthropic 推出的 AI 工具扩展接口标准，让 AI Agent 能直接调用外部数据源和工具。API 是更传统的数据接入方式。

对 PE/VC 投研来说，**MCP/API 的价值不是"技术炫酷"，而是"信息获取提效"**——以前需要手动打开 5 个网站查的数据，现在可以让 AI Agent 一次调完。

本 Skill 按投研工作流的 **7 大信息需求场景**组织 MCP/API 选型：

| 场景 | 核心需求 | 本 Skill 对应章节 |
|------|----------|-------------------|
| 政策/法规检索 | 中国政策、法律条文、监管动态 | §二 A 类 |
| 学术/前沿检索 | 论文、临床试验、技术突破 | §二 B 类 |
| 企业/工商检索 | 公司股权、融资、高管、诉讼 | §二 C 类 |
| 金融/市场数据 | 股票行情、研报、SEC 文件 | §二 D 类 |
| 新闻/舆情检索 | 实时新闻、行业动态、社媒监控 | §二 E 类 |
| 网页抓取/工具 | 通用搜索、网页提取、浏览器自动化 | §二 F 类 |
| **科技/工程/标准** 🆕 | **芯片产品数据、科学计算验证、技术标准、AI 模型生态** | **§二 G 类** |

每个 MCP/API 列出：**覆盖范围、投研适用场景、安装/接入方式、是否免费、与交付能力系列的对应关系**。

**BioMCP 投研亮点**（为什么单独展开）：

```
核心优势：跨实体跳转（Cross-Entity Pivots）

biomcp gene drugs BRAF        → 从靶点基因查相关药物
biomcp gene trials BRAF        → 从靶点基因查相关临床试验
biomcp disease drugs melanoma  → 从疾病查治疗药物
biomcp drug adverse-events pembrolizumab → 查药物不良事件信号
biomcp search all --gene BRAF --disease melanoma → 一键跨实体全景扫描

投研场景：
- "这家 CGT 公司的靶点 BRAF V600E，有哪些在研药物和临床试验？" → 一条命令
- "GLP-1 药物的 FDA 批准状态、不良事件、短缺情况？" → 一条命令
- "某种癌症的发病率/生存率数据？" → SEER Explorer
- "某个靶点的科研资助情况？" → NIH Reporter
```

**ArXiv MCP Server 投研亮点**（科技前沿的完整链条）：

```
核心优势：论文研究全流程闭环

search_papers  → 搜索（布尔查询+分类过滤+日期范围）
download_paper → 下载（HTML优先，PDF回退；分页安全读取）
read_paper     → 阅读（Markdown输出+分页）
citation_graph → 引用图谱（Semantic Scholar 引用+被引，无需本地下载）
watch_topic    → 主题监控（定期检查新论文，仅返回增量）
check_alerts   → 查看提醒
semantic_search → 语义搜索（本地论文集合上的语义匹配）
deep-paper-analysis → 深度分析提示词（从方法论到实践影响的系统评估）
compare_papers  → 多篇论文技术对比

投研场景：
- "量子纠错最新突破在哪？" → search_papers + watch_topic 持续监控
- "这个技术路线是不是主流？" → citation_graph 量化引用影响力
- "KAN vs Transformer 投资怎么选？" → compare_papers 系统对比
- "这家公司创始人的论文实际影响力？" → citation_graph + deep-paper-analysis
- "本周新出了哪些 LLM 推理加速论文？" → check_alerts
```

**HuggingFace MCP 投研亮点**（AI/ML 投资的全能 MCP）：

```
核心优势：AI 生态全景查询

模型搜索 → 查 AI 模型（下载量/架构/任务类型/库依赖）
数据集搜索 → 查数据集（标签/规模/作者）
Spaces 语义搜索 → 找 AI 应用（自然语言描述 → 匹配 Gradio 应用）
论文语义搜索 → 找 ML 研究论文（语义搜索而非关键词）
文档语义搜索 → 查 HF 文档（API 参考和教程）
社区工具 → 运行 MCP 兼容的 Gradio Spaces（图像生成/音频处理等）
Jobs → 在 HF 基础设施上运行/监控任务

投研场景：
- "这个 AI 公司说用了 LLaMA，下载量趋势怎样？" → 模型搜索
- "AI 视觉语言模型论文有哪些？" → 论文语义搜索
- "这个技术有没有开源替代？" → Spaces 语义搜索
- "LLM 微调最好的 PEFT 方案？" → 文档语义搜索
```

**与交付能力对应**：B 类 MCP 直接增强 `02_论文与前沿动态检索方法论.md`——原来需要手动查 arXiv/PubMed/顶会，现在 AI 可以一键调。**BioMCP 是 `08_生物医药_CGT投资框架.md` 的最佳 MCP 伴侣**；**ArXiv MCP + HuggingFace MCP 是 `09_AI_Native_SaaS投资框架.md` + `16_半导体_芯片投资框架.md` + `14_量子科技投资框架.md` 的最佳 MCP 伴侣**。

```
│   → 判断产品覆盖广度 vs 专注深度
├── 技术参数对比：搜索同类芯片的 Datasheet
│   → 判断国产替代的性能差距（"好用" vs "能用"）
├── Errata 清单：已知缺陷和修正记录
│   → 判断产品成熟度（Bug 多 = 还在"能用"阶段）
├── 新产品发布节奏：按时间追踪产品线迭代
│   → 判断技术迭代速度
└── 参考设计数量：某产品线有多少参考设计
    → 判断生态成熟度（参考设计多 = 客户易采用）

G2 Wolfram Alpha MCP —— 技术可行性验证场景
├── 新能源：验算 LCOE（度电成本）、光伏/风电效率极限
├── 半导体：验算芯片制程物理极限（摩尔定律边界）
├── 核聚变：验算等离子体参数（温度/压力/约束时间 Lawson 判据）
├── 量子计算：验算退相干时间 vs 门操作时间
├── 合成生物：验算代谢路径效率/理论产率上限
├── 材料：验算力学性能/热导率/电导率理论极限
└── 通用：任何"这个技术理论上可行吗"的验证

G4/G5 GitHub + HuggingFace —— AI/SaaS 投资场景
├── 查开源项目活跃度：Issue/PR 频率、贡献者数量、Reactions 情绪
├── 查 AI 模型下载量趋势：HuggingFace 模型搜索
├── 查竞品技术栈：代码搜索 + 依赖关系分析

└── 查技术趋势：新兴技术在 GitHub 上的采纳曲线
```

## 六、红旗与注意事项

1. ⚠️ **MCP 不是万能的**——数据质量取决于上游数据源。BioMCP 的 PubMed 数据质量高，但 Google Scholar MCP 搜索结果质量需人工筛选
2. ⚠️ **付费 MCP 需评估 ROI**——Wind/天眼查/北大法宝都是年费几万，小基金未必需要全部接入
3. ⚠️ **合规边界**——通过 MCP 查 SEC 文件合法，但直接联系竞品现任员工访谈不合法；MCP 提供数据，不代表数据使用合规
4. ⚠️ **API Key 安全**——不要把 API Key 写在公开的配置文件里；用环境变量或 .env 管理
5. ⚠️ **MCP 稳定性**——开源 MCP 可能更新频繁或中断；关键投研流程建议保留手动备份
6. ⚠️ **专利 MCP 缺口**——目前没有成熟的专利检索 MCP（智慧芽/USPTO/EPO 的 API 是付费的，还没封装成 MCP）；这是 nei-pevc 可以推动的方向
