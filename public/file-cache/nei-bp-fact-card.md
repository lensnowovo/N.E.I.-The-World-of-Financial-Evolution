# BP 快速拆解与项目事实卡

## 任务目标

把一份 BP、CIM、teaser、项目介绍或融资材料，转换成一张便于投资团队继续工作的项目事实卡。

本 Skill 只回答：

1. 这家公司是谁，在做什么？
2. 它处于什么行业和产业链位置？
3. 产品卖给谁，为什么有人付钱？
4. 材料披露了哪些经营、财务、团队和融资事实？
5. 哪些说法有材料支持，哪些只是公司口径或分析推断？
6. 材料还缺什么，下一轮应该问什么？

本 Skill **不回答**：

- 是否符合某家机构或基金的投资范围；
- 是否值得投资、应该投多少钱；
- 估值是否合理；
- 未经外部核验的市场规模、客户、订单或技术结论是否真实。

这些判断应在事实卡完成后，结合机构策略、基金约束、历史项目和外部证据单独进行。

## 隐私与材料边界

- 只在用户明确授权的本地或可信 Agent 客户端中处理项目材料。
- 不要求把 BP、财务数据、客户名单或访谈记录上传给 N.E.I.
- 如果材料包含个人身份证号、联系方式、银行账号、底层客户名单或未公开交易条款，先提醒用户脱敏。
- 不向输出中复述与分析无关的个人敏感信息。

## 输入

至少提供以下一种材料：

- BP / Pitch Deck；
- CIM / teaser；
- 项目一页纸或融资介绍；
- 官网、产品手册、会议纪要等材料的组合；
- 用户粘贴的项目摘要。

可选信息：

- 材料名称、版本和日期；
- 公司所属地区；
- 当前融资轮次；
- 用户特别想核对的问题。

如果材料无法完整读取，先列出：

- 已读取的文件和页码范围；
- 无法识别、缺页、乱码或图表模糊的位置；
- 本次事实卡的覆盖范围。

不要在材料残缺时假装已经读完。

## 核心纪律

### 1. 事实状态必须分开

所有重要信息使用以下状态之一：

| 状态 | 含义 |
|---|---|
| **材料披露** | BP 或其他输入材料直接写明 |
| **计算结果** | 根据已披露数字进行简单、可复算的计算 |
| **合理推断** | 根据材料作出的分析推断，不等于事实 |
| **外部已验证** | 只有实际调用外部来源并给出出处时才能使用 |
| **未披露** | 材料没有提供，不能自行补全 |
| **存在冲突** | 不同页面、表格或材料的说法不一致 |

不得把“合理推断”写成“材料披露”，不得把公司自述写成第三方已验证事实。

### 2. 重要信息必须可追溯

- PDF 或幻灯片：标注页码，例如 `BP 第 12 页`。
- 无页码文档：标注章节、标题或原文关键词。
- 多份材料：标注文件名和页码。
- 用户只提供口头摘要：标注 `用户提供，未见原始材料`。

无法定位来源时，写明 `来源位置待确认`，不要编造页码。

### 3. 不强行凑亮点、红旗或数字

- 没有足够证据时，写“材料不足以判断”。
- 不要求固定输出“三个亮点”“三个风险”。
- 不因为 BP 没写某项，就直接判断公司存在问题；先归入“未披露”。
- 不用 1–5 分制造虚假精确感。

### 4. 行业描述与行业验证分开

可以根据材料说明：

- 公司自称属于什么行业；
- 处于产业链哪个位置；
- 服务哪类客户和场景；
- 材料提到了哪些竞品、替代方案和市场驱动。

但除非实际查询了外部可靠来源，否则：

- 不确认 TAM、增速和渗透率真实；
- 不确认公司是否“行业第一”；
- 不确认竞品数据和政策判断；
- 不补写记忆中的市场数字。

## 工作流程

### 第一步：登记材料与读取质量

建立材料清单：

| 材料 | 日期/版本 | 已读取范围 | 读取质量 | 备注 |
|---|---|---|---|---|

识别材料是否陈旧、是否缺少财务附件、是否存在多个版本。

### 第二步：提取公司基本事实

提取并核对：

- 公司全称、简称、成立时间、总部和主要经营地；
- 当前轮次、拟融资金额、资金用途；
- 一句话业务定义；
- 主要产品或服务；
- 核心客户与付费方；
- 当前发展阶段；
- 材料提到的重要资质、专利、牌照或认证。

一句话业务定义使用下面的句式：

> 公司通过【产品/能力】，为【客户】解决【问题】，主要通过【收费方式】获得收入。

句子中缺失的部分直接标记“未披露”。

### 第三步：拆解产品与客户价值

回答：

- 产品具体是什么，而不是宣传口号是什么？
- 产品处于研发、试点、量产还是规模销售阶段？
- 谁使用、谁决策、谁付钱？
- 客户为什么现在需要它？
- 客户不用它时采用什么替代方案？
- 购买、部署和交付周期多长？
- 产品效果用什么指标衡量？
- 是否存在定制化过重、交付依赖人力或产品边界模糊的问题？

涉及技术时，只复述材料能够支持的技术路径、性能指标和当前成熟度，不自行认定“领先”。

### 第四步：定位行业与产业链

形成一张“行业位置卡”：

| 项目 | 材料所述内容 | 状态与来源 |
|---|---|---|
| 行业/子行业 |  |  |
| 产业链环节 | 上游 / 中游 / 下游 / 平台 / 服务 |  |
| 目标市场 |  |  |
| 主要客户类型 |  |  |
| 关键供应方 |  |  |
| 直接竞品 |  |  |
| 间接替代方案 |  |  |
| 需求驱动 |  |  |
| 行业关键约束 |  |  |

另外列出“需要外部验证的行业主张”，例如市场规模、行业增速、竞争排名、政策红利和渗透率。

### 第五步：拆解商业模式与经营进展

提取：

- 收费对象和收费方式；
- 单价、合同期限、回款安排；
- 获客渠道和销售周期；
- 交付方式和售后要求；
- 复购、续费、留存或重复采购；
- 收入结构和客户集中度；
- 在手订单、合同、意向订单的口径差异；
- 产能、利用率、良率或研发里程碑；
- 规模化的主要约束。

对不同行业补充最少必要字段：

- **SaaS / 软件**：ARR、订阅收入、续费、流失、客单价、实施周期、毛利率。
- **消费 / 品牌**：渠道结构、复购、退货、投放、库存、经销商依赖、单店或单品经济性。
- **制造 / 硬科技**：产能、良率、认证、量产、核心设备、供应链、在手订单、客户导入周期。
- **医疗 / 生物科技**：管线阶段、关键终点、注册路径、患者人群、CMC、现金消耗和下一里程碑。
- **平台 / 交易型业务**：GMV 与收入的区别、抽佣率、供需两侧、留存和补贴依赖。

只输出与当前项目相关的补充字段，不机械展示全部行业模板。

### 第六步：统一数据口径

把材料中的关键数字整理为：

| 指标 | 数值 | 期间 | 单位 | 口径 | 同比/变化 | 来源 | 状态 |
|---|---:|---|---|---|---|---|---|

重点检查：

- 累计值与单期值是否混用；
- 含税与不含税、签约与确认收入、订单与回款是否混用；
- GMV、流水、收入和净收入是否混用；
- 用户数、注册数、活跃数、付费数是否混用；
- 预测数是否与实际数放在一起；
- 年化数字是否由短期数据直接外推；
- 百分比的分母是否明确；
- 图表和正文是否一致；
- 不同币种和数量级是否统一。

允许进行简单计算，但必须写出公式，例如：

> 计算结果：2025 年收入同比增长约 25% =（5,000 万 - 4,000 万）/ 4,000 万。

### 第七步：拆解团队、股权与融资

整理：

- 创始人和核心成员的岗位、相关经历及投入状态；
- 团队与当前业务之间的经验匹配；
- 材料披露的员工规模和关键团队缺口；
- 已有股东与历史融资；
- 当前股权结构、员工期权池；
- 本轮融资金额、估值口径、出让比例；
- 资金用途和预计可支撑时间；
- 未来 12–24 个月关键里程碑。

不得根据姓名、年龄、性别、籍贯等信息作不相关评价。团队判断只围绕公开经历、能力覆盖和业务执行证据。

### 第八步：建立主张—证据表

把材料中最影响后续判断的主张放入下表：

| 关键主张 | 支持证据 | 来源 | 状态 | 可信度 | 需要怎样验证 |
|---|---|---|---|---|---|

可信度只用于描述材料支持强弱：

- **高**：有明确数据、合同/审计/第三方材料或多处一致披露；
- **中**：有具体描述或单一数据，但尚无独立佐证；
- **低**：只有宣传性结论、预测或模糊表述。

可信度不是对公司诚信的最终判断。

### 第九步：识别冲突与缺口

分别输出：

1. **材料内部冲突**：数字、时间、客户、产品状态等前后不一致。
2. **定义不清**：指标没有口径或分母。
3. **关键未披露**：理解项目所必需但材料没有提供的信息。
4. **待外部验证**：需要工商、招投标、论文、专利、客户访谈或行业数据验证的说法。
5. **材料质量问题**：过期、缺页、图表不可读、预测和实际混排。

### 第十步：生成下一轮问题

只围绕已经发现的缺口和冲突生成问题，按优先级排序：

- **P0：不回答就无法理解项目**
- **P1：影响商业和经营质量判断**
- **P2：可在后续尽调补充**

每个问题附一行“为什么问”。

避免空泛问题，例如“公司的竞争优势是什么”。优先使用可核实问法，例如：

> BP 第 18 页写 2025 年收入 5,000 万，第 23 页图表显示 4,200 万。两者分别采用什么确认口径？

## 输出格式

# [公司名称]｜BP 快速拆解与项目事实卡

## 0. 阅读范围

- 材料：
- 版本/日期：
- 已读取范围：
- 无法读取部分：
- 本次结论边界：

## 1. 项目速览

| 项目 | 内容 | 状态与来源 |
|---|---|---|
| 一句话业务定义 |  |  |
| 公司与地区 |  |  |
| 当前阶段 |  |  |
| 产品/服务 |  |  |
| 客户与付费方 |  |  |
| 商业模式 |  |  |
| 核心经营进展 |  |  |
| 当前融资诉求 |  |  |

## 2. 公司、产品与客户

用简洁段落说明产品是什么、客户是谁、解决什么问题、如何交付和收费。区分材料披露与合理推断。

## 3. 行业与产业链位置

填写行业位置卡，并列出需要外部验证的行业主张。

## 4. 商业模式与经营进展

说明收费、获客、交付、复购/续费、客户集中度和规模化约束。

## 5. 关键数据表

统一期间、单位和口径；预测数据单独标识。

## 6. 团队、股权与融资

只整理材料支持的事实，并标记未披露项。

## 7. 关键主张—证据表

至少覆盖最影响理解项目的主张；数量由材料决定，不强行凑数。

## 8. 冲突、缺口与待验证事项

按“内部冲突 / 定义不清 / 未披露 / 待外部验证”分组。

## 9. 下一轮问题

按 P0 / P1 / P2 排序，每题说明为什么问。

## 10. 交接说明

用不超过 150 字说明：

- 目前已经能够确认什么；
- 仍然不能确认什么；
- 下一步最值得补哪类材料。

结尾固定注明：

> 本事实卡只整理材料信息，不构成投资建议，也没有判断项目是否符合特定机构或基金的投资要求。

## 输出前自检

提交结果前逐项检查：

- [ ] 是否明确列出读取范围和无法读取部分？
- [ ] 是否给重要事实标注了来源？
- [ ] 是否把材料披露、计算结果、推断和未披露分开？
- [ ] 是否避免自行补充未经验证的行业数字？
- [ ] 是否统一了期间、单位和指标口径？
- [ ] 是否指出了材料内部冲突？
- [ ] 是否没有强行凑亮点、风险或评分？
- [ ] 是否没有给出脱离基金背景的投资建议？
- [ ] 下一轮问题是否来自真实缺口，而不是通用问题模板？

如果任一项不满足，先修正再输出。
