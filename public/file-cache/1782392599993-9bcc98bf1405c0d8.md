# 专家访谈方法论 · Skill 草稿

## 这是什么

面向 PE/VC/FA 投研人员的**专家访谈全流程 SOP**——从专家网络选型、专家筛选、访谈大纲设计、访谈执行到结论整理与交叉验证。一级市场投研中，公开渠道找不到的信息（竞争格局、行业趋势、企业核心竞争力、未公开订单、客户真实反馈）必须通过专家访谈获取。这是一项"按分钟计费"的核心调研技能——做得好能拿到独占洞察，做不好既浪费预算又拿不到有效信息。但很多投研人不知道专家网络怎么用、大纲怎么设计、怎么问隐私问题、怎么识别专家质量、怎么交叉验证。本 Skill 把专家访谈拆成"选专家 → 备大纲 → 执行 → 整理 → 结算验证"五步，可作为任何行业尽调的标准动作。适合一级市场投研、战略咨询、商业研究、市场调研。

## 方法论框架（6 维度）

### 维度 1：专家网络选型（找谁约）

#### 主流专家网络（专家库管理机构）

| 平台 | 特点 | 价格区间 |
|---|---|---|
| **凯盛 Capvision** | 中国市场份额领先 | 中高 |
| **BCC** | 国际化 | 中高 |
| **Third Bridge** | 国际化，高质量 | 高 |
| **GLG** | 全球最大专家网络 | 高 |
| **小型专家公司** | 从大平台跳槽出来的员工创办 | 中低 |

**关键机制**：
- 访谈通过专家网络电话会议系统转接，**双方匿名**——投资人不知专家联系方式，专家不知投资人身份。
- 按分钟/小时计费，时长由系统记录作为结算依据。
- 建议同时向多家专家网络提交需求，避免单家匹配失败时无备选。

#### 访谈成本

- 普通专家：3000-4000 RMB/小时。
- 顶尖专家：6000-8000 RMB/小时。
- 前 15 分钟内终止（专家不符合要求）通常不收费。
- 超过 15 分钟按比例计费（例：20 分钟 = 1/3 小时 × 单价）。

### 维度 2：专家筛选标准（选什么样的）

> 同一个岗位/公司，选对专家和选错专家差距巨大。

#### 4 大筛选标准

##### 1. 在职状态：离职 1-6 个月最佳

- **在职**：受公司保密约束，不愿分享机密信息。
- **离职 1-6 个月**：信息最新鲜 + 顾虑最少 → 最佳。
- **离职 > 1 年**：公司内部机制已变，信息可能过时。

##### 2. 部门/职位：匹配问题

- 问**业务/竞争力**→ 业务/销售出身最合适（天天接触客户，知道炮火往哪打）。
- 问**技术/产品**→ 研发/产品出身。
- 问**财务/运营**→ 财务/运营出身。
- 中后台人员对前台业务竞争力了解不深。

##### 3. 级别：中层 > 高层 > 基层

- **中层**（总监/资深经理）→ 知道细节 + 仍在一线 → 信息密度最高。
- **高层**（VP/CXO）→ 战略视野 + 但细节模糊。
- **基层**→ 视野局限。

##### 4. 机构背景：竞品/上下游/同业

- 目标公司**前员工**（离职 1-6 月）→ 最直接。
- **竞品员工**→ 横向对比视角。
- **上下游客户/供应商**→ 第三方验证。
- **行业协会/研究机构**→ 行业全局视角。

### 维度 3：访谈大纲设计（问什么）

#### 大纲设计原则

1. **问题数量**：10-15 个，时间不超 1 小时（专家超 1 小时疲劳，质量下降）。
2. **前置 1-2 个"试金石"问题**：前 15 分钟内用 1-2 个关键问题测试专家专业度——若不行立即终止（前 15 分钟终止通常不收费）。
3. **结构分层**：行业全局 → 公司细节 → 量化数据 → 判断预测。
4. **备"替代问法"**：隐私/机密问题用替代问法（见下）。

#### 大纲模板（5 段式）

```
1. 开场（2-3 分钟）：介绍访谈目的，建立信任。
2. 试金石（10-15 分钟）：1-2 个核心问题，判断专家质量。
3. 行业与公司（20-25 分钟）：竞争格局、核心竞争力、行业趋势。
4. 量化数据（15-20 分钟）：市场份额、增长率、关键指标（用层层递进法）。
5. 判断与预测（10 分钟）：行业未来、公司风险、投资视角。
```

### 维度 4：提问技巧（怎么问）

> 两类问题最难问：隐私类和定量类——必须用特殊技巧。

#### 技巧 1：隐私/机密问题的替代问法

**无效问法**（直接问隐私）：
- ❌ "XX 公司给客户经理的奖金政策是怎么样的？"
- ❌ "XX 公司的核心风险有哪些？"

**有效问法**（假设性问法）：
- ✅ "如果您来设计一个社交电商公司客户经理的激励机制，您会如何设计，才能提升积极性？"
- ✅ "如果您来设计社交电商的供应商资质审核方案，您会如何处理风险控制？"

专家基于原公司经验回答，但避开了"泄露机密"的顾虑。

#### 技巧 2：定量数据的层层递进法

> 行业内人士往往不记精确数字，但能给出区间——用层层递进引导。

**示例对话**：
```
Q：您认为中国电商行业中最有竞争力的公司有哪些？
A：阿里、京东、拼多多，苏宁也不错。

Q：这四家基本垄断市场了吗？能占 90% 以上吗？
A：差不多，至少有 80% 吧。

Q：这四家里面，谁的份额更高？
A：阿里最大，京东也不错，拼多多和苏宁低一些。

Q：阿里和其他三家差距大吗？
A：挺大的，2018 年数据是远超其他三家之和。

Q：那是否可以假设：阿里 60%，京东 15%，拼多多+苏宁约 5%？
A：差不多。
```

**要点**：不要直接问精确数字——通过"区间 + 对比 + 假设确认"逐步逼近。

#### 技巧 3：跑题及时打断

- 专家易跑题，你按分钟计费，每分钟都是钱。
- 礼貌打断：*"不好意思打断一下，我们还是回到 XX 问题上。"*
- 不要因害羞被带跑——这是付费服务，你有权控场。

#### 技巧 4：交叉验证

- 访谈中随时记录关键数字。
- 专家前后矛盾时立即追问。
- 一个关键数据至少访谈 2-3 个独立专家交叉验证。

### 维度 5：结论整理与交付（怎么用）

#### 结论文档结构

```
# 专家访谈纪要 · [主题] · [日期]

## 核心要点（老板/客户先看这部分）
1. 关键结论 1
2. 关键结论 2
3. 关键结论 3
4. 关键结论 4

## 详细纪要
[完整对话或要点整理]

## 交叉验证
- 与专家 A 的对比：……
- 与公开数据的对比：……

## 待补充
- 访谈中遗漏的问题，后续补充
```

#### 补充问题机制

老板/客户读完报告后常有新问题——可委托专家网络找原专家：
- 文字补充：通常免费或低价。
- 短时访谈（10-15 分钟）：按比例计费。

### 维度 6：质量识别与结算（怎么验）

#### 专家质量识别（前 15 分钟试金石）

| 信号 | 高质量专家 | 低质量专家 |
|---|---|---|
| 具体细节 | 能说人名/数字/时点 | 笼统模糊 |
| 行业认知 | 知道非公开动态 | 只复述公开信息 |
| 自我纠错 | 主动修正/限定 | 一概肯定/夸大 |
| 边界意识 | 明确说"这个我不确定" | 什么都能答 |
| 反例/黑料 | 能讲失败案例 | 只讲好话 |

#### 结算谈判权

- 访谈后做交叉验证，若发现数据破绽/编造，可与专家网络协商减免。
- 案例：发现数据多处破绽 → 协商打五折。

## 行业核心指标

| 指标 | 健康阈值 | 说明 |
|---|---|---|
| **单专家成本** | 3000-6000 RMB/小时 | 视专家级别 |
| **单专家时长** | 1 小时（核心） | 不超 1.5 小时 |
| **关键问题专家数** | ≥ 3 人独立交叉验证 | 单一专家不可信 |
| **试金石通过率** | > 70% | 低于此说明筛选不严 |
| **量化数据获取率** | 关键指标均有区间估计 | 层层递进法 |
| **结论交叉验证** | 与公开数据 + 多专家对比 | 单源不交付 |
| **15 分钟止损** | 不专业专家立即终止 | 避免浪费预算 |

## 红旗识别
1. **未做试金石测试就深聊**——前 15 分钟不筛质量，整场可能浪费。
2. **选在职高管做核心访谈**——保密约束大，拿不到有效信息。
3. **选错部门/职位**——问业务却找产品/技术专家，答非所问。
4. **直接问隐私/机密**——专家直接拒答，浪费时间。
5. **不层层递进问数据**——问精确数字专家答不出，无效。
6. **不控制时间被带跑题**——按分钟计费，被带跑 = 烧钱。
7. **单一专家定结论**——单源风险极高，必须交叉验证。
8. **无文字纪要，只凭记忆**——细节丢失，无法交付。
9. **结论无"核心要点"摘要**——老板/客户没时间读全文。
10. **不做结算谈判**——数据有破绽也不协商，浪费预算。
11. **不追踪补充问题**——老板新需求无法闭环。
12. **专家说"什么都能答"**——低质量信号，真专家有边界意识。

## 占位符
- `[目标公司]`：被研究的公司
- `[访谈主题]`：竞争格局 / 行业趋势 / 技术路线 / 客户反馈
- `[专家类型]`：前员工 / 竞品员工 / 上下游 / 行业协会
- `[专家级别]`：高层 / 中层 / 基层
- `[专家网络]`：凯盛 / BCC / Third Bridge / GLG
- `[试金石问题]`：用于前 15 分钟判断专家质量的关键问题
- `[核心结论]`：3-5 条要交付的关键结论
