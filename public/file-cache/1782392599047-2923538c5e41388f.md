# 论文与前沿动态检索方法论 · Skill 草稿

## 这是什么

面向 PE/VC/FA 投研人员（尤其硬科技/生物医药/AI4S 赛道）的**学术论文与前沿科技动态检索 SOP**。投前沿科技必须看论文——技术路线选型、PI 团队追踪、竞品进展、突破时点判断，论文是最一手的信息源。但很多投资人看论文只会搜 PubMed 或 Google Scholar 关键词，不会追 PI、不会用 arXiv、不懂顶会体系、不会设 alert。本 Skill 把论文检索拆成"选数据库 → 检索 → 追踪作者 → 设 alert → 判断突破"五步，可作为任何前沿科技尽调的前置动作。适合投硬科技/生物医药/AI/新材料/量子/聚变等前沿赛道的投资人、AI4S 研究员、技术战略人员。

## 方法论框架（6 维度）

### 维度 1：数据库选型矩阵（去哪搜）

> 不同领域有不同的核心数据库——选错数据库等于漏掉一半文献。

| 领域 | 首选数据库 | 补充数据库 | 预印本平台 |
|---|---|---|---|
| **AI / 机器学习** | Google Scholar / Semantic Scholar | IEEE Xplore / ACM DL | **arXiv**（绝对核心） |
| **生物医药 / 临床** | **PubMed**（绝对核心） | Web of Science / Embase | bioRxiv / medRxiv |
| **化学 / 材料** | Web of Science / Scopus | ACS / RSC 期刊库 | ChemRxiv |
| **物理 / 量子** | Web of Science / APS | Physical Review 系列 | arXiv |
| **综合 / 跨学科** | **Google Scholar**（最全） | Web of Science（被引权威） | - |

**关键认知**：
- **Google Scholar** 覆盖最广，但质量参差不齐——适合初筛。
- **Web of Science** 的"被引频次"是判断高影响力论文的权威指标——适合找里程碑式突破。
- **arXiv/bioRxiv** 是抢占首发权的平台——AI 和生物医药的最新进展往往先出现在这里，比正式发表早 6-12 个月。
- **PubMed** 是生物医药的绝对核心——投生物医药/CGT 必须熟练使用。

### 维度 2：顶会与顶刊体系（看哪些）

> 论文的"分量"取决于发表在哪里——投研要识别顶会顶刊，才能判断技术突破的真实分量。

#### AI / 机器学习顶会（CCF-A 类最权威）

| 会议 | 领域 | 分量 |
|---|---|---|
| **NeurIPS** | 综合机器学习 | 最高 |
| **ICML** | 机器学习 | 最高 |
| **ICLR** | 深度学习 | 最高 |
| **CVPR / ICCV / ECCV** | 计算机视觉 | 最高 |
| **ACL / EMNLP** | NLP | 最高 |
| **AAAI / IJCAI** | AI 综合 | 高 |
| **KDD** | 数据挖掘 | 高 |

**实战技巧**：CV Paper Portal、Papers with Code 等聚合平台可一站式检索顶会论文。

#### 生物医药顶刊

| 期刊 | 分量 |
|---|---|
| **Nature / Science / Cell**（CNS） | 最高 |
| NEJM / Lancet / JAMA（医学临床） | 最高 |
| Nature Medicine / Nature Biotech / Nature Genetics | 极高 |
| Cell Research / Cell Reports Medicine | 高 |

#### 综合性顶刊

Nature / Science / PNAS 是跨学科最高刊物——硬科技突破常首发于此。

### 维度 3：检索技巧（怎么搜）

#### 关键词策略

- **关键词组合**：核心词 + 同义词 + 缩写。例如 "CAR-T" OR "chimeric antigen receptor" OR "CAR T cell"。
- **限定时间**：只看近 1-2 年，快速定位前沿。
- **限定领域**：用 MeSH（PubMed）/ IPC 排除不相关文献。
- **被引阈值**：在 Web of Science 设被引 > 50/100，过滤高影响力论文。

#### 检索进阶技巧（投研场景）

```bash
# 场景 1：找某技术路线最新进展（PubMed）
"CAR-T" AND "solid tumor" AND ("2023"[Date - Publication] : "2025"[Date - Publication])
sorted by: Most Recent

# 场景 2：找高被引里程碑论文（Web of Science）
"CRISPR" AND "gene editing"
sorted by: Citations: high to low

# 场景 3：找 arXiv 上最新 AI 预印本
cs.AI category, last 30 days, keyword: "large language model"
```

#### 工具辅助

- **Connected Papers** / **Research Rabbit**：输入一篇论文，自动生成关联论文图谱——快速定位一个领域的论文群。
- **Semantic Scholar**：AI 驱动的学术搜索，自动提炼摘要、识别 influential citations。
- **Papers with Code**：AI 领域，论文 + 开源代码 + SOTA 榜单三合一。

### 维度 4：追踪作者/PI（追谁）

> 在前沿科技领域，**追人比追关键词更重要**——一个 PI 的研究方向往往引领一个子领域。

#### 追踪路径

1. **找核心作者**：从高被引论文的通讯作者/第一作者切入。
2. **查作者主页**：
   - Google Scholar Author Profile（被引总数、h-index、最新论文）
   - ResearchGate（社交、问答、未发表数据）
   - 实验室官网（PI 的团队、在研项目、招生方向）
3. **追踪作者动态**：
   - Google Scholar Alert（作者发表新论文时邮件提醒）
   - ResearchGate Follow
   - 实验室官网 RSS（如有）
4. **判断 PI 影响力**：
   - h-index / i10-index（学术影响力）
   - 近 3 年被引趋势（是否仍在巅峰）
   - 是否连续在顶会顶刊发表（持续输出能力）
   - 是否有产业合作 / spin-off（商业化能力）

#### 投研场景：判断科学家创业项目

- 查 PI 的 Google Scholar → 看论文质量与持续输出
- 查 PI 的专利（见专利检索 Skill）→ 看 IP 是否已布局
- 查 PI 的 spin-off 历史 → 看是否有商业化经验
- 查 PI 的团队规模与博士生去向 → 看人才梯队

### 维度 5：alert 与持续监测（怎么跟）

> 论文检索不是一次性动作，要建立持续监测体系。

#### Alert 设置清单

| 平台 | Alert 类型 | 设置建议 |
|---|---|---|
| **Google Scholar Alert** | 关键词 + 作者 | 每个核心关键词 + 每个追踪 PI 各设一个 |
| **PubMed Alert（My NCBI）** | 关键词检索式 | 投生物医药必备 |
| **arXiv RSS / 邮件订阅** | 分类订阅 | cs.AI / q-bio / cond-mat 等分类 |
| **顶会官网** | 论文发布通知 | NeurIPS / ICML / CVPR 等 |
| **期刊官网** | Current Issue TOC | Nature / Science / Cell 等 |

#### 监测节奏

- **日级**：arXiv / bioRxiv 预印本（快速抢占）
- **周级**：Google Scholar Alert 邮件汇总
- **月级**：PubMed / 期刊 TOC
- **年度**：顶会论文集（NeurIPS 12 月、ICML 7 月、CVPR 6 月）

### 维度 6：判断"真突破" vs " incremental"（怎么判）

> 论文检索的终极目标不是"找到论文"，而是**判断哪些是真正的技术突破**。

| 判断维度 | 真突破 | Incremental |
|---|---|---|
| **发表平台** | 顶会顶刊（CNS / NeurIPS / Nature 子刊） | 低影响力期刊 |
| **被引增速** | 发表 1 年内被引 > 100 | 长期低位 |
| **follow-up 数量** | 发表后 6-12 个月内大量 follow-up | 无 follow-up |
| **代码/数据开源** | 开源且被广泛复现 | 无开源 |
| **媒体/社区讨论** | Hacker News / Twitter 学术圈热议 | 无讨论 |
| **PI 背景** | 该领域公认领军人物 | 不知名团队 |
| **产业关注** | 被头部公司引用或跟进 | 无产业动作 |

**投研场景应用**：判断一个科学家创业项目时，用上表对 PI 的代表作打分——若多项符合"真突破"，技术风险显著降低。

## 行业核心指标

| 指标 | 健康阈值 | 说明 |
|---|---|---|
| **数据库覆盖** | 至少 2 个核心数据库 | 避免单一来源偏差 |
| **顶会顶刊识别能力** | 准确判断论文分量 | 判断技术突破真实性 |
| **作者追踪深度** | PI 的 h-index + 近 3 年趋势 | 判断学术影响力 |
| **Alert 覆盖** | 关键词 + 作者 + 顶会 + 期刊 | 持续监测 |
| **预印本监测** | arXiv/bioRxiv 日级关注 | 抢占提前量 |
| **被引分析** | Web of Science 被引 > 50/100 | 高影响力论文识别 |
| **突破判断** | 7 维度打分 | 区分真突破 vs incremental |

## 红旗识别
1. **只会用百度搜论文**——覆盖极不完整，必须用专业数据库。
2. **不分顶会顶刊，把所有论文当同等分量**——低影响力期刊的"突破"水分极大。
3. **不看预印本（arXiv/bioRxiv）**——AI/生物医药最新进展往往在预印本。
4. **不追作者，只追关键词**——前沿领域的方向常由少数 PI 引领。
5. **不会设 alert，靠临时搜**——错过首发窗口。
6. **只看摘要不看方法**——摘要宣称的突破和实际方法可能差距巨大。
7. **不看 follow-up**——一篇"突破"若无 follow-up，可能是死胡同。
8. **不查 PI 的 spin-off 历史**——PI 是否有商业化经验是创业项目关键。
9. **被引数作为唯一指标**——被引高不等于真突破（可能是有争议研究）。
10. **不区分理论突破 vs 工程突破**——理论突破离商业化可能很远。
11. **不查专利**——论文公开的技术可能已被专利保护（见专利检索 Skill）。
12. **过度相信单篇论文**——需多篇独立研究验证才是真突破。
